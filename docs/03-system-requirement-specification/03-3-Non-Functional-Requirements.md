백엔드 시스템의 **성능, 안정성, 보안**에 대한 품질 기준을 정의한다. (포트폴리오 핵심 어필 포인트)

### [NFR-01] 성능 및 응답 속도 (Performance)

- **REQ-PERF-01**: 캐시(Redis)에 존재하는 단어 조회 및 검증은 **50ms 이내**에 응답해야 한다.
- **REQ-PERF-02**: AI 검증(External API)이 필요한 경우, 클라이언트 타임아웃 방지를 위해 **비동기 처리**하거나 **3초 이내**에 처리 결과를 반환해야 한다.
- **REQ-PERF-03**: 그래프 조회 시, 한 번에 로딩하는 노드 개수는 **100개 이하**로 제한(Pagination/Lazy Loading)하여 브라우저 렌더링 부하를 최소화한다.

### [NFR-02] 동시성 및 데이터 무결성 (Concurrency)

- **REQ-CONC-01**: 동일한 부모 노드에 대해 다수의 유저가 동시에 단어를 입력할 경우, **선착순 1명**의 요청만 처리되고 나머지는 반려되어야 한다. (Redis Distributed Lock 적용)
- **REQ-CONC-02**: 스트림 탈취 발생 시, 기존 스트림의 상태 변경(Active -> Archived)과 새 스트림 생성이 **원자적(Atomic)**으로 처리되어야 한다.

### [NFR-03] 보안 (Security)

- **REQ-SEC-01**: 모든 API 통신은 HTTPS 상에서 이루어져야 한다. (배포 시 적용)
- **REQ-SEC-02**: 게임 내 욕설 및 비속어 필터링을 적용해야 하며, AI 프롬프트 주입(Injection) 공격을 방어해야 한다.
